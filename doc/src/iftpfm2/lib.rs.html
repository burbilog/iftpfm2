40// Dependencies that were in main.rs and are used by multiple modules,
41// or are fundamental to the library's operation, can be listed here
42// or within the specific modules that use them.
43// For now, each module handles its own specific imports.
44// Common ones like `std::io`, `std::process` are used directly in modules.
45// External crates like `ftp`, `regex`, `chrono`, `once_cell`, `rayon`, `ctrlc`, `scopeguard`
46// will need to be listed in Cargo.toml and then `use`d in the modules that need them.
47
48// Example of how PROGRAM_NAME and PROGRAM_VERSION might be used from within a module if not passed:
49// use crate::{PROGRAM_NAME, PROGRAM_VERSION};
50// This is now handled by making them pub const in lib.rs and modules using `crate::PROGRAM_NAME`.
51
52// The `main` function in `src/main.rs` will now primarily use items from this library crate.
53// e.g. `use iftpfm2::config::parse_config;` or `use iftpfm2::*;`
54//
55// We'll need to ensure that all necessary `pub` keywords are used within each module
56// for items that need to be accessed by other modules or by `main.rs` via this lib.rs.
57// For example, `Config` struct in `config.rs` must be `pub struct Config`.
58// Functions like `parse_config` must be `pub fn parse_config`.
59// Statics like `LOG_FILE` in `logging.rs` must be `pub static LOG_FILE`.
60
61// Note: The temporary const declarations for PROGRAM_NAME in cli.rs and instance.rs
62// were removed, and those modules now use `crate::PROGRAM_NAME` as intended.
